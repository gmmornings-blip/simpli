# Simpli Design System

> **Version:** 0.1.0

---

# Purpose

The Simpli Design System provides reusable UI components and visual rules that create a consistent experience across every module.

---

# Design Tokens

## Spacing

* xs
* sm
* md
* lg
* xl
* xxl

Use spacing tokens consistently.

---

## Border Radius

* small
* medium
* large

Rounded, but not exaggerated.

---

## Typography

Levels:

* Display
* Heading
* Title
* Body
* Caption

Readable before decorative.

---

# Colors

Semantic colors only.

* Primary
* Secondary
* Success
* Warning
* Error
* Info
* Neutral

Avoid assigning meaning through arbitrary colors.

---

# Icons

Use one icon library across the platform.

Icons support text—they do not replace it.

---

# Components

Shared components include:

* Button
* Input
* TextArea
* Checkbox
* Radio
* Select
* Card
* Modal
* Drawer
* Table
* Badge
* Tabs
* Toast
* Avatar
* Navigation
* Breadcrumb
* Search Bar
* Calendar
* Data Grid

These components belong to Simpli Core and should not be duplicated by modules.

---

# States

Every interactive component should support:

* Default
* Hover
* Focus
* Active
* Disabled
* Loading
* Error

---

# Accessibility

All components must:

* support keyboard navigation,
* include accessible labels,
* provide sufficient contrast,
* expose focus states.

Accessibility is a core requirement.

---

# Responsiveness

Components adapt gracefully to:

* Desktop
* Tablet
* Mobile

No separate mobile design language.

---

# Final Principle

A new module should look like Simpli on its first day simply by using the shared design system.
